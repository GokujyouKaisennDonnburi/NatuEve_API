# ADR-0002: メール送受信の責務分離とサブドメイン戦略

- ステータス: Accepted
- 日付: 2026-07-07
- 関連: #92, ADR-0001, ADR-0003

## コンテキストと課題

メールの「受信」と「送信」を同一ドメインの同一設定で扱うと、
送信の到達性やスパム判定（送信レピュテーション）の変動が受信側にも波及し、
問題の切り分けや DNS 認証（SPF/DKIM/DMARC）の管理が複雑になる。
責務ごとに使うサービスとドメインを分けて、影響範囲を隔離したい。

## 決定

メールの送受信を責務ごとに分離し、ドメインを使い分ける。

- **受信**: **Cloudflare Email Routing** を **ルートドメイン**で運用する。
- **送信**: **Resend**（ADR-0001）を **`mail.` サブドメイン**で運用する。

## 理由

- **レピュテーション隔離**: 送信ドメインの評価変動を受信・ルートドメインから切り離す。
  送信で問題が起きてもルートドメインのメール受信や他用途に波及しない。
- 送受信で DNS レコード（MX / SPF / DKIM / DMARC）の責務が明確に分かれ、
  設定ミスや相互干渉を防げる。

## 影響（結果）

### 良い影響
- 送信トラブルの影響がサブドメインに閉じ、切り分けが容易になる。
- 送信・受信それぞれのサービスを独立して差し替えられる（ADR-0001 の移行方針と整合）。

### トレードオフ・制約
- **DMARC はサブドメイン単位で管理する**。ルートと `mail.` サブドメインで
  ポリシーを別々に設計・運用する必要がある。
- DNS レコードが増え、初期セットアップとドキュメント整備の手間が増える。

## 検討した代替案

- **単一ドメインで送受信を兼ねる**: 設定はシンプルだが、送信レピュテーションの
  変動が受信・ルートに直撃するため不採用。
