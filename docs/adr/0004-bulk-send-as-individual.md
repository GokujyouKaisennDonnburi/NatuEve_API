# ADR-0004: 一斉送信は個別送信で行う（特定電子メール法対応）

- ステータス: Accepted
- 日付: 2026-07-07
- 関連: #92, ADR-0001

## コンテキストと課題

複数ユーザーへ同じ内容のメール（運用上のお知らせ等）を送る場面がある。
このとき `To` や `BCC` に全員をまとめて入れると、宛先が相互に露出したり
（`To` の場合）、一斉配信としての性質が強まり、法令・プライバシー上の懸念が生じる。
日本の**特定電子メール法**（特定電子メールの送信の適正化等に関する法律）は
広告宣伝メールにオプトインを求めるため、どのメールが規制対象になるかの整理も必要。

## 決定

一斉送信は**受信者ごとの個別送信**で行う。`To`/`BCC` に全員を入れる方式は取らない。

あわせて、**運用上の通知（お知らせ・障害連絡・仕様変更周知など）はトランザクションメール
として扱う**。広告宣伝を主目的とするメールとは明確に区別する。

## 理由

- **プライバシー保護**: 個別送信により、受信者間でメールアドレスが露出しない。
- **特定電子メール法への適合**: 運用通知はサービス提供に付随する取引・連絡目的の
  メールとして扱い、広告宣伝メール（オプトイン規制の対象）と切り分ける。
  広告宣伝目的のメールを送る場合は、別途オプトイン取得・オプトアウト導線・
  送信者表示等の要件を満たす設計を行う（本 ADR の対象外）。

## 影響（結果）

### 良い影響
- 宛先の相互露出事故を構造的に防げる。
- メールの目的（運用通知 / 広告宣伝）が分類され、法令対応の判断がしやすくなる。

### トレードオフ・制約
- 受信者数ぶん送信 API を呼ぶため、**送信通数・レート**が増える
  （ADR-0001 の 100 通/日制約に直結する）。大量送信時はバッチ・レート制御を設計する。
- 「運用通知」と「広告宣伝」の線引きは実運用で判断が要る。判断に迷う内容は
  広告宣伝寄りとみなし、オプトイン要件を満たしてから送る。

## 検討した代替案

- **`BCC` に全員を入れて 1 通で送る**: 送信通数は減るが、一斉配信色が強く、
  誤送信時の被害が大きい。個別送信の利点（露出防止・目的分類）を得られないため不採用。
- **`To` に全員を入れる**: 宛先が相互に露出するため論外。不採用。
